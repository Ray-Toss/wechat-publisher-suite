# 微信公众号格式规范

本文档记录微信公众号平台的格式要求和最佳实践。

## 基础格式要求

### 1. HTML标签要求
❌ 禁止使用Markdown语法，必须使用纯HTML标签
✅ 所有元素必须带有内联样式，避免外部CSS

### 2. 图片规范
- 必须使用HTML `<img>` 标签，禁止使用`![alt](url)`格式
- 图片必须先上传到微信素材库，使用微信返回的URL
- 禁止使用外部图片链接，避免被微信屏蔽
- 图片样式推荐：
```html
<p style="text-align: center; margin: 20px 0;">
  <img src="微信图片URL" 
       style="max-width: 100%; height: auto; display: block; margin: 0 auto; border-radius: 8px; box-shadow: 0 2px 12px rgba(0,0,0,0.1);" 
       alt="图片描述"/>
</p>
```

### 3. 文本样式规范
| 元素 | 样式要求 |
|------|----------|
| 标题H1 | 字体22px，加粗，居中，上下边距30px/20px |
| 标题H2 | 字体18px，加粗，左边框4px蓝色，上下边距25px/15px |
| 标题H3 | 字体17px，加粗，上下边距20px/10px |
| 正文段落 | 字体16px，行高1.8，首行缩进2em，上下边距15px |
| 引用块 | 背景浅灰，左边框4px，内边距15px，斜体 |
| 加粗文本 | 红色突出显示 |
| 代码块 | 深色背景，浅色文字，内边距20px，圆角8px |
| 行内代码 | 浅灰背景，内边距2px 6px，圆角3px |
| 分隔线 | 1px浅灰色实线，上下边距30px |

### 4. 长度限制
- 标题：最长64字符
- 摘要：最长120字符
- 正文：建议2000-6000字
- 图片数量：建议3-8张

## 内容规范
1. 段落长度：每段3-5句，适合手机阅读
2. 章节划分：每800-1000字一个章节，配一张图
3. 重点内容：使用加粗突出，不要整段加粗
4. 交互引导：结尾可以提问互动，但不要诱导分享

## 常见问题
### 图片不显示
- 检查是否使用了外部图片链接
- 确认图片已经上传到微信素材库
- 检查图片URL是否有效

### 排版错乱
- 避免使用复杂的CSS样式
- 不要使用浮动、定位等布局属性
- 所有样式使用内联style属性

### 发布失败
- 检查内容是否包含敏感词
- 确认公众号授权有效
- 检查API_KEY是否正确配置
